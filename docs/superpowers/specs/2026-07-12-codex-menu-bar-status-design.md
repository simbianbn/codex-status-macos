# Codex Menu Bar Status — Design Specification

## Goal

สร้างแอป macOS Menu Bar แบบ Native สำหรับแสดงโควตา Codex ที่เหลืออยู่และสถานะการทำงานล่าสุด โดยผู้ใช้เห็นข้อมูลย่อจากแคปซูลบน Menu Bar และคลิกเพื่อดูรายละเอียดใน Popover ได้

## Scope

- แอปทำงานบน macOS และแสดงผลผ่าน Menu Bar โดยไม่ต้องมีหน้าต่างหลัก
- อ่านโควตาจริงจากข้อมูล usage/session ที่ Codex CLI หรือ Codex Desktop บันทึกไว้ในเครื่อง
- ติดตามสถานะงานเป็น `ว่าง`, `กำลังทำงาน`, `เสร็จ`, หรือ `เกิดข้อผิดพลาด`
- รีเฟรชอัตโนมัติทุก 30 วินาที และรีเฟรชทันทีเมื่อผู้ใช้เปิด Popover
- ไม่คำนวณหรือสร้างเปอร์เซ็นต์จำลองเมื่อไม่มีข้อมูลจริง

## Menu Bar Capsule

แคปซูลแสดงข้อความ `Codex 72%` เมื่อพบโควตาที่ตรวจสอบได้ และแสดง `Codex —` เมื่อไม่พบข้อมูล

สีพื้นหลังตามโควตาคงเหลือ:

- เขียว: มากกว่า 50%
- เหลือง: 20–50%
- แดง: ต่ำกว่า 20%
- เทา: ไม่พบข้อมูล, ยังไม่เข้าสู่ระบบ, หรือข้อมูลหมดอายุ

จุดสถานะด้านซ้ายของข้อความ:

- เทา: ว่าง
- ฟ้าพร้อม animation แบบ pulse: กำลังทำงาน
- เขียว: งานล่าสุดเสร็จสำเร็จ
- แดง: งานล่าสุดเกิดข้อผิดพลาด

เพื่อให้ Menu Bar อ่านง่ายบนพื้นหลังหลายแบบ แอปต้องรองรับ Light/Dark appearance และคง contrast ของข้อความตามมาตรฐานระบบ

## Detail Popover

เมื่อคลิกแคปซูล ให้แสดง Popover ที่ประกอบด้วย:

1. เปอร์เซ็นต์โควตาคงเหลือและ progress bar
2. ช่วงโควตาที่ตรวจพบและเวลาที่จะรีเซ็ต หากแหล่งข้อมูลระบุ
3. สถานะงานล่าสุด
4. ชื่องานหรือคำอธิบายสั้น เวลาเริ่ม และเวลาเสร็จ หากมี
5. เวลาที่อ่านข้อมูลล่าสุด
6. ข้อความสาเหตุเมื่ออ่านข้อมูลไม่ได้
7. ปุ่ม `รีเฟรช` และ `ออกจากแอป`

Popover ต้องเปิดได้ทันทีโดยไม่รอการอ่านข้อมูลรอบใหม่ และค่อยอัปเดตเมื่อการอ่านเสร็จ

## Architecture

### App shell

ใช้ Swift และ SwiftUI โดยใช้ `MenuBarExtra` เป็นจุดเริ่มต้นของ UI และแยก View ย่อยสำหรับแคปซูล, usage summary, task status และ error state

### Usage provider

`CodexUsageProvider` มีหน้าที่ค้นหาและอ่านข้อมูล usage/session ในเครื่องแบบ read-only แล้วคืนค่า normalized model ซึ่งประกอบด้วยเปอร์เซ็นต์ที่เหลือ, เวลาที่รีเซ็ต, เวลาอัปเดต และแหล่งที่มาของข้อมูล

Provider ต้องตรวจสอบรูปแบบข้อมูลอย่างระมัดระวัง รองรับกรณี field หายหรือ schema เปลี่ยน และไม่ส่งค่าที่คาดเดาให้ UI หากยืนยันเปอร์เซ็นต์ไม่ได้

### Activity provider

`CodexActivityProvider` สรุปสถานะจาก session/task events ล่าสุด โดย mapping event ที่กำลังดำเนินการเป็น `working`, event สำเร็จเป็น `completed`, event ล้มเหลวเป็น `failed` และไม่มีงานที่กำลังทำเป็น `idle`

สถานะ `completed` และ `failed` จะแสดงเป็นสถานะงานล่าสุดจนกว่าจะเริ่มงานใหม่ ผู้ใช้จึงมองเห็นผลลัพธ์ได้ ไม่ถูกเปลี่ยนเป็น `idle` ทันที

### Status store

`StatusStore` ทำงานบน Main Actor รวมผลจาก providers ดูแล timer 30 วินาที และเผยแพร่ view state ให้ SwiftUI การอ่านไฟล์และ parsing ทำงานนอก main thread เพื่อไม่ให้ Popover กระตุก

## Data Safety

- อ่านเฉพาะ metadata ที่จำเป็นต่อ usage และสถานะ
- ไม่แสดงหรือบันทึก prompt, response, token, API key หรือข้อมูลลับ
- ไม่แก้ไขไฟล์ของ Codex
- ไม่ส่งข้อมูลออกนอกเครื่อง
- log ของแอปต้องไม่มีเนื้อหางานหรือ credential

## Error Handling

- ไม่พบ Codex data: แสดงแคปซูลสีเทาและคำแนะนำให้เปิด/เข้าสู่ระบบ Codex
- schema ไม่รองรับ: แสดงว่าไม่สามารถอ่านรูปแบบข้อมูลปัจจุบัน พร้อมปุ่มรีเฟรช
- permission denied: แสดง path กลุ่มข้อมูลที่ต้องอนุญาตโดยไม่เปิดเผยเนื้อหา
- ข้อมูลเก่า: แสดงข้อมูลล่าสุดพร้อมป้าย `ข้อมูลอาจล้าสมัย` และเวลาที่อ่านได้
- การอ่านรอบหนึ่งล้มเหลว: เก็บค่าล่าสุดที่ยืนยันแล้วไว้ แต่แสดงสถานะ stale อย่างชัดเจน

## Testing and Acceptance Criteria

- Unit tests ครอบคลุม threshold สีที่ 19%, 20%, 50%, 51% และ unknown
- Unit tests ครอบคลุม parsing ของข้อมูลปกติ, field หาย, malformed data และ schema ที่ไม่รองรับ
- Unit tests ครอบคลุมสถานะ idle, working, completed และ failed
- Popover เปิดทันทีแม้ provider กำลังอ่านข้อมูล
- เมื่อมี usage ที่ตรวจสอบได้ แคปซูลและ Popover แสดงเปอร์เซ็นต์เดียวกัน
- เมื่อไม่มี usage ที่ตรวจสอบได้ UI ต้องแสดง `—` และไม่แสดงเปอร์เซ็นต์ที่สร้างขึ้นเอง
- แอปไม่อ่านหรือแสดงเนื้อหา prompt/response และไม่เขียนไฟล์ Codex
- Build และ test ผ่านบน macOS target ที่กำหนดในโปรเจกต์

## Out of Scope

- การแก้ไขหรือเพิ่มโควตา Codex
- การเรียก billing API หรือประมาณโควตาจากค่าใช้จ่าย
- การรองรับ Windows/Linux
- การแสดงประวัติ prompt หรือ response
- Browser automation และการ scrape หน้าเว็บ
- การเผยแพร่ผ่าน Mac App Store ในรอบแรก
